Plant-Based Recipe Web Application

Through this project, I gained significant insights into full-stack development, including the challenges of integrating frontend and backend components as well as working with 3rd party APIs. I identified and addressed gaps in my web development knowledge, enhancing my understanding of the end-to-end workflow. The experience allowed me to refine my skills in frontend development, particularly in creating a user-friendly interface and ensuring seamless functionality. This project not only deepened my technical expertise but also fueled my passion for developing solutions that support healthier and more sustainable living.

  • Objective

    The objective of this web application was to develop a centralized platform for plant-based recipe inspiration, designed to simplify and promote plant-based eating. By offering a user-friendly interface and a diverse range of over 600 recipes, the goal was to make plant-based cooking more accessible and enjoyable for a broader audience.

    Design and Development Process

    Design

    The design process began with selecting a suitable API for recipe data retrieval. I opted for the Spoonacular API due to its extensive database and robust features. I used Postman to practice making API calls and understand the JSON object format, ensuring smooth integration with the application.

    With the API set, I focused on designing the user interface using Angular:

    • User Authentication: Created a login form enabling users to log in or create new accounts. Account creation involves answering dietary preference questions and setting a username and password, providing a personalized experience.
    • Recipe Widget: Developed a widget to display recipe images, titles, and types. The widget includes a button for detailed recipe views and a heart icon for liking and saving recipes. This encourages user interaction and personalization.
    • Saved Recipes List: Implemented a feature for users to access and manage their liked recipes, allowing them to view, save, and remove items from their list. This enhances user engagement by offering easy access to their favorite recipes.

    The application framework was designed with more features in mind, however, due to end of semester business, I had to pair down in complexity. That being said, I am working on adding the following enhancements:

    • Filtering Feature: Adding a filter option to refine recipe searches based on cost, cuisine, dietary restrictions, etc.
    • Personalized Dashboard: Creating an algorithm that considers user cooking/dietary preferences to generate a personalized dashboard, offering a tailored recipe experience.
    • Tried Recipes Feature: Developing a feature to track and display recipes a user has already tried, further personalizing the application.

    Development

    The development phase involved building both the frontend and backend components to implement the designed features:

    • Frontend Development: Utilized Angular to create the login system, interactive recipe widget, and saved recipes list. Integrated API calls for dynamic recipe data retrieval and ensured a responsive, user-friendly interface.
    • Backend Development: Established a simply yet robust backend with three main entities:
      • User Entity: Manages user data with a unique username and user ID as the primary key, facilitating secure authentication.
      • Recipe Entity: Stores recipe information from Spoonacular, with a recipe ID as the primary key, allowing efficient data management.
      • Liked List Entity: Used to implement a many-to-many relationship between users and recipes by linking unique recipe ID to a unique user ID, enabling dynamic interaction and personalization.
